XFAIL list (documented failures)
--------------------------------

Each line: h2spec test id → short reason. Fixes are tracked as
follow-ups under the "HTTP/2 edge-case polish" milestone
(post-compliance-gate, not blocking Step 10 benchmark work).

Remaining failures all live below our own code — they're nghttp2
library-level defaults or a fundamental architectural trade-off (dual-
protocol listener). Fixing each means either forking/patching nghttp2
or restructuring the port-level protocol routing; neither is worth it
for edge cases no real client hits.

1. generic/2/5 — Sends a PRIORITY frame on closed stream.
   RFC says we MUST accept PRIORITY on closed streams. nghttp2's
   default behaviour is to ignore → no response frame → h2spec
   times out waiting for a subsequent PING-ACK. Not workload-
   relevant: real clients never send PRIORITY frames (we don't
   support RFC 9218 priority; see §1.2).

2. generic/3/8/1 — FIXED 2026-04-22. Manual PING-ACK submission in
   cb_on_frame_recv bypasses nghttp2's session_is_closing auto-ACK
   gate (nghttp2#1103). Also closes http2/7/1 (same root).

3. http2/3.5/2 — FIXED 2026-04-22. Phase 1 of the protocol-registry
   plan (PLAN_PROTOCOL_REGISTRY.md) made detection handler-aware:
   addHttp2Handler without addHttpHandler now means "h2-only", and
   non-preface garbage routes to the h2 bad-preface path which emits
   the hand-crafted GOAWAY(PROTOCOL_ERROR).

4. http2/4.2/2 — Large DATA frame exceeding SETTINGS_MAX_FRAME_SIZE.
   Expected: GOAWAY(FRAME_SIZE_ERROR). Actual: echo DATA back.
   nghttp2 silently caps oversized frames at the decode layer;
   mitigated in practice by our MAX_FRAME_SIZE=16384 SETTINGS we
   advertise upfront.

5-7. http2/5.1/{11,12,13} — DATA/HEADERS/CONTINUATION on closed
    stream. Expected: connection error STREAM_CLOSED. Actual:
    timeout. Rare — well-behaved clients never do this.

8. http2/5.1.1/2 — FIXED 2026-04-22 (Group B). on_begin_frame gate
   enforces §5.1.1 stream-id rules (odd, monotonic, reuse-of-closed)
   and submits GOAWAY(PROTOCOL_ERROR) before nghttp2's own (softer)
   stream-level RST_STREAM handling kicks in.

9. http2/5.1.2/1 — HEADERS exceeding MAX_CONCURRENT_STREAMS.
   Expected: RST_STREAM(PROTOCOL_ERROR | REFUSED_STREAM). Actual:
   HEADERS response. nghttp2 rejects silently at the connection
   layer; the MCS=100 setting prevents this from being a real
   problem.

10. http2/6.7/4 — FIXED 2026-04-22. h2 strategy now closes the TCP
    channel when both want_read and want_write fall to 0 after the
    drain, instead of leaving the peer hanging on EOF timeout. The
    GOAWAY frame still precedes the close (nghttp2 auto-submitted it
    on invalid-PING detection); h2spec accepts that ordering as long
    as the close is prompt.

11. http2/7/1 — FIXED 2026-04-22 together with #2.
